配置个人博客过程实录

1、github网页：

​ 新建Repositories，其项目名应与github用户名相同，名称为wiirong.github.io

2、安装 git （清华镜像下载）

3、安装 node.js （官网下载即可，下载LTS版）

4、使用cmd（WIN+R召出并运行“cmd”）

​ 【改变 cmd 的位置：先输入盘符和冒号， 再输入d盘内目标文件名（路径） 即可（不区分大小写）】

5、验证git和nodejs的安装

【上述这些命令行结果都显示为各自的版本号就可以了】

6、安装 hexo

（本文这些代码都是在 cmd 中的 D：/Blog位置下运行的）

然后用 hexo -v 检查版本，显示版本号就行

7、配置blog文件

在cmd中d：/blog 下输入：hexo init

耐性等待下载即可

8、生成本地主页

继续输入：

第一个是生成静态文件，第二个是启动服务器（localhost:4000/）

（可以本地查看博客，复制快捷键会关闭本地预览，不必关闭，再开一个cmd即可）

9、将本地文件上传到github仓库

（1）配置SSH KEY

​ 在cmd中d：/blog输入：

​ 再创建ssh key：

​ ssh-keygen -t rsa -C "903590780@qq.com"

​ 会询问几个问题（放置文件位置、密码等），enter键跳过即可

（2）前往上述放置文件位置，用文本打开.pub文件，全选复制其中代码。

（3）打开github个人主页的setting，进入“SSH and GPG keys”选项，新建一个SSH KEY ，名字随便起为 blog，把刚才复制的公钥粘贴进去。

（4）在命令行继续输入：

​ ssh -T git@github.com

​ 检查是否能连接

10、将hexo部署到github

（1）打开blog文件夹中的 _config.yml 文件（文本打开）

找到最下面的deploy，修改并添加，结果如下（两个单引号代表略，替换成git即可）：

​ 这个文件里还可以修改网站相关信息（标题、副标题、语言、作者等等）

（2）再cmd输入：

​ npm install hexo-deployer-git --save

（3）再输入：

​ hexo clean

​ hexo generate

​ hexo deploy

​ 上述三个分别为了：清除缓存、更新文件、部署

​ *部署deploy的时候会弹框要求输入github的账号和密码。

11、检查能否登录个人博客

​ https://wiirong.github.io/

12、发表文章：

（1） 在blog文件夹中的 source / _posts 里面创建“.md”文件（可以直接在blog文件夹中右键启动git Bash并输入 hexo new “文章名” 来完成这一步骤。）

（2）利用CMD 且在d盘的blog目录下，输入老三样（后两行的第二个单词可以简写成一个字母g、d。另：clean不能缩写，但如非原地更新，则无需clean。）：

​ hexo clean

​ hexo generate

​ hexo deploy

clean清除缓存这一步如非必须则可以省略不写。

（3）用批处理.bat简化发表步骤（弄不成，懒得再试了）
